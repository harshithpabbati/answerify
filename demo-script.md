# Answerify 3-Minute Demo Script

## 0:00–0:20 — Problem
"Support teams waste time on repetitive emails, context switching between docs and tools, and inconsistent replies when ticket volume spikes. Answerify solves that with an AI support workspace that drafts grounded responses and automates routine workflows."

## 0:20–0:45 — Setup + Ingestion
"In onboarding, we create an organization workspace, connect the support inbox, and add documentation sources. Answerify indexes this content into a searchable knowledge base so every reply can be grounded in real company context."

## 0:45–1:10 — Inbox + Intent Detection
"Now in the shared inbox, inbound emails are automatically tagged by intent—like billing, bugs, onboarding, privacy, or refunds. This gives immediate triage and routes each conversation into the right handling path."

## 1:10–1:35 — Retrieval + Draft Generation
"For each conversation, Answerify runs retrieval over indexed sections, expands nearby context, and generates an HTML draft response using the most relevant knowledge."

## 1:35–1:55 — Citations and Trust
"Each draft includes citations, so agents can verify exactly where the answer came from before sending. That keeps responses accurate, auditable, and safer for customer-facing support."

## 1:55–2:20 — Autopilot Confidence Logic
"Answerify combines retrieval and model confidence to decide action: high-confidence replies can be auto-sent, while lower-confidence replies are saved as drafts for human review."

## 2:20–2:40 — Workflows + MCP Augmentation
"Teams can define support workflows and policies, and Answerify can call MCP-connected tools for live business data before replying—so responses are both context-aware and operationally current."

## 2:40–3:00 — Learning Loop + Close
"When agents approve and edit strong replies, Answerify feeds that back into the knowledge loop so future responses improve over time. Net result: faster support, more consistent quality, and less manual triage overhead."
